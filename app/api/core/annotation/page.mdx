# **Annotation Object Overview**

## **Annotation Attributes**

Annotations can include additional **human-judged metadata** that enhances labeled data.  
For example, you might want to **measure occlusion levels** of vehicles in an image.

To achieve this, we support **annotation_attributes**, an object that defines additional properties for each annotation.

### **Supported Use Cases**

✅ **Define categorical attributes** (e.g., parked or not).  
✅ **Capture numerical values** (e.g., speed, distance).  
✅ **Specify angles** (e.g., object orientation).  
✅ **Allow free text input** (e.g., object description).  
✅ **Create relationships between annotations** (e.g., linking two objects).

---

## **Annotation Attribute Types**

Each attribute must specify a **type**, determining how data is collected.  
If no type is provided, it defaults to a **Categorical Attribute**.

## **Request Format**

To create a **task with attributes**, include the **annotation_attributes** parameter in your API request.

---

### **Example: JSON Payload for Image Annotation**

```json filename="/tasks/imageannotation.json"
{
  "callback_url": "http://www.example.com/callback",
  "instruction": "Draw boxes around the vehicles in the image.",
  "attachment_type": "image",
  "attachment": "http://i.imgur.com/v4cBreD.jpg",
  "geometries": {
    "box": {
      "objects_to_annotate": ["car", "pedestrian"]
    }
  },
  "annotation_attributes": {
    "parked": {
      "description": "Is the car currently parked?",
      "choices": ["Yes", "No"]
    },
    "heading": {
      "description": "Which direction is the car heading?",
      "choices": ["left", "right", "back", "front"],
      "conditions": {
        "label_condition": {
          "label": "car"
        },
        "attribute_conditions": [
          {
            "parked": "No"
          }
        ]
      }
    }
  }
}
```
