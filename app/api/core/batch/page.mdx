# **Batches Object Overview**

## **What are Batches?**

For **high-volume projects**, **batches** help organize and manage **large-scale tasks** by dividing them into **smaller units**. Batches can:

- **Group tasks** based on **dataset types**.
- **Organize work by submission periods** (e.g., weekly submissions).
- **Track task progress** and improve **workflow efficiency**.

---

## **Batch Workflows in Skala**

### **1Ô∏è‚É£ Skala Enterprise Workflow**

Designed for **large-scale enterprise projects** with **ongoing task submissions**.

**Process Flow:**

1. **Batch is created** ‚Üí `status = in_progress`
2. **Tasks are added to the batch** ‚Üí Specify the `batch` field on the task using the **batch name**.
3. **Once the final task is completed**:
   - The **batch is marked as completed** ‚Üí `status = completed`
   - A **callback is fired** to notify completion.

---

### **2Ô∏è‚É£ Skala Rapid Workflow**

For **real-time data processing**, Skala Rapid provides a more **structured** approach.

üöÄ **Key Differences in Skala Rapid:**

- **Two types of batches:**
  - **Production Batches** ‚Üí Used for normal labeling tasks.
  - **Calibration Batches** ‚Üí Used to **test and validate** the labeling process **before full-scale production**.

**Process Flow:**

1. **Batch is created** ‚Üí `status = staging`
2. **Tasks are added to the batch** ‚Üí Specify the `batch` field.
3. **Batches must be finalized** ‚Üí `status = in_progress`
4. **Once finalized:**
   - Tasks are **submitted for labeling**.
   - **No new tasks can be added** to the batch.
5. **When the final task is completed**:
   - The batch is **marked as completed** ‚Üí `status = completed`
   - A **callback is fired** for confirmation.

---

### **3Ô∏è‚É£ Skala Studio Workflow**

For **internal team-based labeling**, Skala Studio ensures **controlled processing**.

**Process Flow:**

1. **Batch is created** ‚Üí `status = staging`
2. **Tasks are added to the batch** ‚Üí Specify the `batch` field.
3. **Batches must be finalized** ‚Üí `status = in_progress`
4. **Once finalized:**
   - Tasks are **sent to your internal team** for labeling.
   - **No new tasks can be added** to the batch.
5. **When the final task is completed**:
   - The batch is **marked as completed** ‚Üí `status = completed`
   - A **callback is fired** for confirmation.
